# Customer evidence cited in the post

A focused reference of the customer-validated evidence quoted in [What are the key benefits of transitioning to agentic coding for software development?](https://claude.com/blog/key-benefits-transitioning-agentic-coding) (2025-12-01). Reproduce these as written when building a pitch.

## Augment Code — timeline acceleration

- **Stack note:** Augment Code uses Claude on Google Cloud's Vertex AI.
- **Outcome:** an enterprise customer completed in **two weeks** a project that their CTO estimated would require **four to eight months** with traditional development.
- **Quote attributed to Guy Gur-Ari, Chief Scientist, Augment Code:**

> "Tasks that would take weeks for a developer to learn can now be completed in a day or two."

- **Why it lands:** anchors the otherwise abstract "acceleration" claim in a single concrete project comparison and a senior technical voice.

## Augment Code — model upgrade quote

The post also includes:

> "Claude Sonnet 4.5's intelligence is immediately noticeable—it makes better use of Augment's codebase context, handles longer-horizon tasks, and opens up new agentic possibilities we're actively exploring."

Use this when the audience is interested in the maturation of the underlying model rather than the workflow.

## Grafana — democratized access

- **Product:** a Claude-powered intelligent assistant for observability data.
- **What it does:** lets users ask natural-language questions about latency, errors, etc. The assistant identifies relevant metrics, constructs PromQL and LogQL queries, correlates data across systems, and presents actionable insights.
- **Example query the post highlights:** "What's causing latency spikes in the checkout service?"
- **Why it lands:** the capability to author PromQL/LogQL was previously gated to engineers with that specific expertise. Agents reduce that gating to a typed question.

## Cross-cutting takeaways the post draws

- The hiring strategy shifts from "rare combinations of expertise" to "strong fundamentals + agent-bridged specialization."
- Tasks once "too expensive" become feasible and technical debt resolves incrementally — useful when arguing for project-economics-level wins.
- Teams "compete effectively against larger competitors despite smaller teams" — useful for fast-moving small teams pitching against incumbents.

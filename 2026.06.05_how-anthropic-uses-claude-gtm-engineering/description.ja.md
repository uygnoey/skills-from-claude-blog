# ある営業担当が Claude Code で GTM ワークフローを作り直した方法

## この記事について
プログラミング経験のなかった営業オペレーターが、Claude Code を使って社内の GTM ツールを作り、改善していった事例です。Gmail 内で顧客メール返信の下書きを作るアプリから始まり、日次ブリーフ／日次リキャップの自動化、さらにスキルとコネクタを束ねたチーム向けプラグインへと拡張していきます。

## どんなときに役立つか
- 顧客メール返信やミーティング後のフォローなど、反復的なコミュニケーション量が多いとき
- エンジニアリング余力が限られる中で、素早く自動化を反復改善したいとき
- スキルと連携（コネクタ）をまとめ、組織全体がすぐ使える形で配布したいとき

## 主なポイント
- まずは最も痛い反復作業を一点突破で解決し、価値が出たら拡張する。
- ドキュメント／CRM／カレンダー／メールなどのソースから、その場で最新コンテキストを取り込む。
- システムプロンプトを何度も調整して文体を合わせ、不要な癖（長さ、過度な慎重さ）を減らす。
- 関係性に応じたトーンのバリエーションを用意する。
- プラグインとして製品化し、チーム導入を加速する。

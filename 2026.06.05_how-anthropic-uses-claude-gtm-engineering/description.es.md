# Cómo un vendedor reconstruyó flujos de trabajo de GTM con Claude Code

## ¿De qué trata este post?
Un caso práctico sobre cómo una persona de ventas sin experiencia previa programando usó Claude Code para crear y mejorar herramientas internas de GTM: empezó con una app dentro de Gmail que redacta borradores de respuesta, y luego la amplió a briefs/recaps diarios y a un plugin para el equipo que empaqueta habilidades y conectores.

## ¿Cuándo es útil?
- Cuando tu equipo gestiona grandes volúmenes de comunicación repetitiva (respuestas por email, seguimientos tras reuniones).
- Cuando necesitas automatizar e iterar rápidamente con poco soporte de ingeniería.
- Cuando quieres empaquetar flujos internos (skills + integraciones) para que otros los adopten en minutos.

## Puntos clave
- Empieza resolviendo un único flujo repetitivo y doloroso; amplía después de demostrar valor.
- Trae contexto “en vivo” desde sistemas fuente (docs, CRM, calendario, email) en el momento de generar.
- Itera mucho sobre el prompt del sistema para ajustar la voz y reducir rasgos no deseados (longitud, excesiva cautela).
- Usa variantes de tono según el tipo de relación.
- Productoízalo como plugin para acelerar la adopción en la organización.

## Recursos incluidos
- Skill: `sales-workflow-briefing` (patrones para briefing/recap y agregación de contexto)
- Guía: “Automatización de flujos GTM con Claude Code”
- Estilo de salida: “concise-sales-email-draft” (borradores breves y seguros)

# Automatización de flujos GTM con Claude Code

## Qué cubre esta guía
Esta guía resume un método repetible inspirado en el post: empezar con un flujo estrecho, traer contexto “en vivo”, iterar prompts para ajustar la voz, y luego empaquetar el resultado para que el equipo lo adopte fácilmente.

## Método
1. **Elige un flujo doloroso** (p. ej., respuestas por email).
2. Define las **fuentes de contexto** que usarás en el momento de ejecutar (email, calendario, CRM, docs).
3. Redacta un **prompt de sistema** y itera hasta que el resultado refleje la voz del equipo.
4. Añade **variantes de tono** según el tipo de relación.
5. Amplía a flujos adyacentes (brief pre-llamada, recap diario) cuando el primero esté estable.
6. **Empaqueta para adopción** (skills + integraciones/conectores) para que otros lo instalen en minutos.

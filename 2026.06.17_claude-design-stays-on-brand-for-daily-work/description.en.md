# Claude Design now stays on brand for daily work

## What is this post?

A product update announcing improvements to Claude Design: stronger design-system adherence, smoother workflow with Claude Code, direct canvas editing, and more export/connector destinations.

## When is it useful?

Useful if your team wants an end-to-end design-to-code workflow that starts from existing design systems and stays consistent across projects, including handoff to Claude Code without starting from screenshots.

## Key points

- Import one or multiple design systems (from GitHub repo, design files, or uploads) and have Claude build with components and auto-correct against the system.
- Use /design-sync to pull a design system into Claude Design, and /design in Claude Code to create/edit/sync designs from the terminal.
- Export reliably to PDF/PowerPoint and send work to connected tools (Adobe, Canva, Miro, Replit, Vercel, Wix, etc.).

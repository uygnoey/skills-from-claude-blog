# Build Day lessons you can reuse

## What this guide covers

A checklist of build-and-verify practices surfaced in the Build Day winners post.

## Checklist

- Define a success demo and a minimal test suite early.
- Plan the work (light PRD + ticket board) before you start coding.
- Split into parallel workflows and integrate late.
- If correctness matters, keep an evidence chain from sources to outputs.
- Add independent verification passes in fresh contexts.
- If a naive approach is expensive, look for clustering/batching strategies and re-validate accuracy.
- Use Claude not only to write code, but also to research tools and integrate unfamiliar technologies.

# Dreaming + memory review checklist

Use this checklist after a set of runs to decide what should be remembered, refined, or removed.

## What to capture
- Recurring mistakes and how to prevent them
- Preferred workflows (ordering of tools / steps)
- User-specific preferences (tone, formats, constraints)
- Workarounds for tricky tool/filetype behavior

## What to avoid
- One-off details that will become stale quickly
- Sensitive information that should not persist

## Output
- A short set of candidate memories (bullet points) with a brief rationale for each.

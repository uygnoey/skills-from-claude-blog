# Launch partners by category — verbatim from the post

The post does not publish a per-vendor category mapping. The buckets below are organized for the reader's convenience using each vendor's commonly understood product category as of the post's publication, but the source post itself only lists the partners as a single alphabetical list. Treat category placement as orientation, not as an Anthropic-asserted classification.

## What the post says about how integrations work

- Powered by the **Claude Compliance API**.
- Data types exposed:
  1. **Conversation content from Claude Enterprise** — chats, uploaded files, projects.
  2. **Activity events across both Claude Enterprise and the Claude Platform** — user logins, admin actions, configuration changes.
- For organizations already using one of the launch partners, the onboarding sentence is: *"connect and configure your Claude instance, and the data flows into the same dashboards and alerting workflows you use for everything else."*

## What the post does not say (and we will not invent)

- Per-vendor entitlement requirements.
- Tier requirements for Claude Enterprise vs. Claude Platform.
- Payload schema, polling vs. push semantics, retention windows, region availability, or residency guarantees.
- Pricing or SLAs.

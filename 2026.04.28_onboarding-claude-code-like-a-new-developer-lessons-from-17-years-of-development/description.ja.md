## この記事について
この記事は、Brendan MacLean（University of Washington の MacCoss Lab）が、新しい開発者を大規模で長寿命なコードベースにオンボーディングするのと同じ考え方で、Claude Code をオンボーディングする方法を紹介します。ポイントは、コンテキストとプロセス文書を「プロジェクトの成果物」として扱い、バージョン管理しながら育てることです。

## どんなときに役立つか
- 暗黙知が多い大規模リポジトリで、Claude Code に効率よく作業させたいとき
- チームが頻繁に入れ替わる環境（学生、オープンソース貢献者など）で、再現性のあるオンボーディングが必要なとき
- 反復のたびに背景説明をやり直すコストを減らすため、永続的でバージョン化されたコンテキストをコードと一緒に管理したいとき

## 主なポイント
- 「AI 用コンテキスト」用の別リポジトリ（記事の例は `pwiz-ai`）を用意し、ブランチの変動に左右されずにガイダンスを蓄積・改善します。
- リポジトリ直下に `CLAUDE.md` を置き、環境構築と追加ドキュメントへの導線をまとめます（“embed ではなく reference”）。
- 繰り返し出てくる作業パターン（例：デバッグ、バージョン管理の規約、プロジェクト概観）を小さなスキルライブラリとして整備します。
- 連携（記事では MCP サーバー）を使って、テスト結果・例外・サポートスレッド・リリースタグなどのプロジェクトデータに Claude Code がアクセスできるようにします。

## 同梱リソース
- 記事にダウンロード可能な添付はなく、`CLAUDE.md` や補助リポジトリ（`pwiz-ai`）といったプロジェクト成果物が例として言及されています。

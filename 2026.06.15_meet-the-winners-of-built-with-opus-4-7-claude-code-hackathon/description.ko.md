## 이 글이 뭔가요
이 글은 Claude의 “Built with Opus 4.7” 해커톤 수상작들을 소개하며, 각 팀이 무엇을 만들었고 Claude 제품(Claude Code, Claude Managed Agents 포함)을 어떻게 활용했는지 요약합니다.

## 언제 유용한가요
- Claude Code 세션 분리와 멀티 에이전트 워크플로우를 통해 실제로 프로토타입을 만드는 사례가 필요할 때
- 여러 팀이 공통으로 강조한 기획(스펙/플랜)·평가·디버깅 방식에서 재사용 가능한 패턴을 찾고 싶을 때

## 핵심 포인트
- 여러 팀이 구현 전에 스펙과 계획을 먼저 세우고, Claude를 단순 코딩 도구가 아니라 사고 파트너로 활용하는 접근을 강조합니다.
- 작업을 병렬화하는 패턴이 반복해서 등장합니다(여러 Claude Code 세션으로 책임을 분리하거나, 디버깅 시 여러 에이전트를 동시에 실행).
- 한 팀은 진단마다 도메인 특화 JSON 지식 파일을 주입해 평가 점수가 상승했다고 보고합니다.

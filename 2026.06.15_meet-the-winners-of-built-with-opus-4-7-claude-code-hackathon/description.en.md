## What is this post?
This post spotlights the winners of Claude’s “Built with Opus 4.7” hackathon, describing what each team built and how they used Claude products (including Claude Code and Claude Managed Agents).

## When is it useful?
- When you want concrete, real-world examples of how builders use Claude Code sessions and multi-agent workflows to ship prototypes.
- When you want patterns for planning, evaluation, and iterative debugging from multiple teams.

## Key points
- Multiple teams emphasize planning-first workflows (writing specs and plans before implementation) and using Claude as a thought partner.
- Several projects describe parallelization patterns: separating work into multiple Claude Code sessions and/or running multiple agents in parallel during debugging.
- One team reports improving evaluation results by injecting a domain-specific JSON knowledge file into every diagnosis.

None. (The post is primarily a set of project spotlights and quotes.)

## ¿De qué trata este post?
Este post presenta a los ganadores del hackathon “Built with Opus 4.7” de Claude, describiendo qué construyó cada equipo y cómo usaron productos de Claude (incluidos Claude Code y Claude Managed Agents).

## ¿Cuándo es útil?
- Cuando quieres ejemplos concretos y reales de cómo la gente usa sesiones de Claude Code y flujos multi‑agente para entregar prototipos.
- Cuando buscas patrones de planificación, evaluación y depuración iterativa aprendidos de varios equipos.

## Puntos clave
- Varios equipos enfatizan trabajar “primero el plan”: escribir especificaciones y planes antes de implementar, y usar a Claude como socio de pensamiento.
- Se repiten patrones de paralelización: separar el trabajo en varias sesiones de Claude Code y/o ejecutar varios agentes en paralelo durante la depuración.
- Un equipo reporta mejoras en su evaluación al inyectar un archivo JSON de conocimiento específico del dominio en cada diagnóstico.

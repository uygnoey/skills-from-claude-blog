## What is this guide?
A short guide to recurring build and iteration patterns described by multiple winners of Claude’s “Built with Opus 4.7” hackathon.

## When is it useful?
Use this when you want a lightweight playbook for planning, parallelization, evaluation, and iteration.

## Key points
- Plan first (spec + plan), then implement.
- Separate components into clean contexts (sessions) and/or use multiple agents in parallel for debugging.
- Ask Claude to audit existing work before moving on.
- Consider eval-first workflows and domain-specific reference data.

## この記事について
この記事は、Claude の「Built with Opus 4.7」ハッカソンの受賞者を紹介し、各チームが何を作り、Claude Code や Claude Managed Agents などの Claude 製品をどのように活用したかをまとめています。

## どんなときに役立つか
- Claude Code のセッション分割やマルチエージェントのワークフローで、実際にプロトタイプを作った具体例を知りたいとき
- 複数チームが共有している計画（仕様/プラン）・評価・デバッグの進め方から再利用できるパターンを探したいとき

## 主なポイント
- 多くのチームが、実装前に仕様と計画を固めること、そして Claude を単なるコーディング支援ではなく思考パートナーとして使うことを強調しています。
- 並列化のパターンが繰り返し登場します（複数の Claude Code セッションに責務を分割、またはデバッグ時に複数エージェントを並列実行）。
- あるチームは、診断ごとにドメイン特化の JSON 知識ファイルを注入することで評価結果が改善したと報告しています。

# Product management on the AI exponential

## What is this post?
This post explains how product managers can adapt their workflow to rapid, compounding improvements in model capability, using Claude Code’s team practices as concrete examples.

## When is it useful?
When you’re building AI products (or using AI tooling) where the feasible solution space changes quickly, making long, fixed roadmaps brittle.

## Key points
- Exponential model progress breaks the assumption that what’s possible at project start is similar to what’s possible at launch.
- A useful workflow split: chat collaboration (Claude.ai) vs. agentic coding (Claude Code) vs. knowledge work (Cowork).
- Four shifts: plan in short sprints (side quests), prefer demos/evals over docs, revisit features with new models, and do the simple thing that works.
- Use prototypes and evals to make discovery cheap and measurable; optimize for capability first (don’t cut tokens too early).

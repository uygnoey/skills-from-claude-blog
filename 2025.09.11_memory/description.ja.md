# Bringing memory to teams

## この記事について
This post announces Memory in the Claude app, which lets Claude remember project context and preferences so teams don't need to re-explain work every chat.

## どんなときに役立つか
Useful when you have ongoing, multi-step work (e.g., sales cycles, sprint planning, long-running initiatives) where continuity and consistent context improve speed and quality, and when you need optionality via incognito chats.

## 主なポイント
- Memory is optional and controlled by the user; you can view and edit what Claude remembers via a memory summary in Settings.
- Claude maintains separate memory per project to keep contexts from mixing and to help contain sensitive information within project boundaries.
- Incognito chats do not appear in history and do not add to memory, providing a “clean slate” for sensitive or one-off conversations.
- To start, enable Memory in Settings and optionally have Claude generate memory from past chats during initial setup.
- You can prompt Claude to update what it focuses on or ignores, and it will adjust the memories it references.

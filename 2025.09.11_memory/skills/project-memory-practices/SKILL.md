# project-memory-practices

## Instructions

- Apply the post’s guidance as product usage practices (not API behavior): treat memory as optional and user-controlled, and keep sensitive work scoped appropriately.    - Prefer project-scoped context: if you are working in multiple initiatives, keep each initiative in its own project so remembered context does not mix.    - Use incognito chats when you do not want a conversation to appear in history or add to memory.    - When memory is enabled, periodically review and edit the memory summary, and tell Claude what to focus on or ignore.

## Examples

- "We have two parallel initiatives. Summarize what you remember for Project A only, and do not use context from other projects."    - "This is a one-off confidential brainstorm—please treat this as an incognito-style conversation and don’t carry details forward."    - "Here is what I want you to stop remembering about our process: …"

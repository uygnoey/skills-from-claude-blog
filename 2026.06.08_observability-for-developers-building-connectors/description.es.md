# Observabilidad para desarrolladores que crean conectores

## ¿De qué trata este post?
Este post anuncia un nuevo panel de observabilidad para conectores publicados en el directorio de Claude y la posibilidad de enviar un servidor MCP al directorio directamente desde la aplicación.

## ¿Cuándo es útil?
- Cuando eres propietario u operador de un conector publicado y necesitas entender la adopción, la fiabilidad y el rendimiento a través de las superficies de producto de Claude.
- Cuando quieres un flujo más sencillo para enviar tu servidor MCP al directorio de conectores.

## Puntos clave
- Los conectores publicados en el directorio ahora incluyen un panel para monitorear su rendimiento en los productos de Claude.
- El panel ayuda a seguir la adopción (usuarios activos, llamadas totales a herramientas, ranking en el directorio a lo largo del tiempo).
- El panel ayuda a diagnosticar problemas (puntuación de salud, tasas de error, latencia y desglose de errores por herramienta).
- Se puede desglosar el uso por producto (p. ej., Claude, Claude Code, Cowork).
- El acceso requiere un plan Team o Enterprise con permisos de Admin/Owner (o un rol personalizado de Enterprise con el permiso Libraries).

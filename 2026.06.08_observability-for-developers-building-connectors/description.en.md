# Observability for developers building connectors

## What is this post?
This post announces a new observability dashboard for published connectors in the Claude directory, plus the ability to submit an MCP server to the directory directly in-app.

## When is it useful?
- When you own or operate a published connector and need to understand adoption, reliability, and performance across Claude product surfaces.
- When you want a smoother workflow to submit your MCP server to the connector directory.

## Key points
- Published directory connectors now have a dashboard to monitor performance across Claude products.
- The dashboard helps track adoption (active users, total tool calls, directory rank over time).
- The dashboard helps diagnose issues (health score, error rates, latency, and per-tool error breakdowns).
- Usage can be broken down by product (e.g., Claude, Claude Code, Cowork).
- Access requires Team or Enterprise with Admin/Owner (or a custom Enterprise role with Libraries permission).

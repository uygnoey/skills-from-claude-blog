# 에이전틱 인터페이스의 진화: Claude Managed Agents로 빌드하기

## 이 글이 뭔가요
이 글은 Claude Managed Agents(프로덕션급 에이전트를 빌드·배포하기 위한 조합형 API 모음)를 소개합니다.

또한 2023년의 “tokens in, tokens out” 방식에서 Claude Code와 Claude Agent SDK 같은 에이전틱 제품/도구로 진화해 온 흐름을 짚고, 에이전트 하네스(두뇌)와 실행 샌드박스(손)를 분리하는 아키텍처로서 Managed Agents를 설명합니다.

## 언제 유용한가요
- 전체 하네스/인프라를 직접 만들고 유지보수하지 않고도 에이전트를 프로덕션에서 안정적으로 운영하고 싶을 때
- 도구 사용 에이전트에 필요한 자격증명 보안, 지속 세션 로그, 관측 가능성, 확장 가능한 오케스트레이션이 필요할 때
- 모델의 추론과 샌드박스 실행을 분리하고 싶을 때(자체 호스팅 샌드박스/VPC 실행 옵션 포함)

## 핵심 포인트
- 프로덕션 에이전트에는 프롬프트 외에도 호스팅/스케일링, 세션 관리, 파일시스템 상태, 실행 격리, 자격증명, 관측 가능성이 필요합니다.
- Managed Agents는 하네스와 샌드박스를 분리하며, 세션은 모델 호출/도구 호출/결과의 append-only 로그로 유지됩니다.
- Vaults와 온디맨드 조회로 자격증명을 샌드박스 밖에 두어, 프롬프트 인젝션에 의한 노출 위험을 줄일 수 있습니다.
- 컨테이너가 준비되기 전부터 추론을 시작할 수 있고, 도구를 쓰지 않는 세션은 컨테이너 기동을 건너뛰어 지연 시간을 줄일 수 있습니다.
- 내구성 있는 세션은 디버깅 타임라인, Memory, Dreaming(세션/메모리를 검토해 미래 메모리를 큐레이션하는 스케줄 프로세스)을 가능하게 합니다.

## 번들 리소스
- Skill: 글에서 도출한 프로덕션 하네스 체크리스트 및 아키텍처 노트
- 레퍼런스 노트: Vaults, 세션, managed/sandbox 분리 개념

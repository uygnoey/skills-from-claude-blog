# エージェンティックな体験の進化: Claude Managed Agentsで構築する

## この記事について
この記事は Claude Managed Agents（プロダクション品質のエージェントを構築・デプロイするための、組み合わせ可能なAPI群）を紹介します。

また、2023年の「tokens in, tokens out」型のAPI利用から、Claude Code や Claude Agent SDK といったエージェンティックな製品へと進化してきた流れを整理し、エージェントのハーネス（「脳」）と実行サンドボックス（「手」）を分離するアーキテクチャとして Managed Agents を説明します。

## どんなときに役立つか
- ハーネスや基盤全体を自前で構築・運用せずに、エージェントをプロダクションで信頼性高く動かしたいとき
- ツール利用エージェントに必要な、認証情報の安全な取り扱い、永続セッションログ、可観測性、スケーラブルなオーケストレーションが必要なとき
- モデルの推論とサンドボックス実行を分離したいとき（セルフホストのサンドボックス/VPC実行の選択肢を含む）

## 主なポイント
- プロダクションのエージェントにはプロンプトだけでなく、ホスティング/スケーリング、セッション管理、ファイルシステム状態、実行分離、認証情報、可観測性が必要です。
- Managed Agents はハーネスとサンドボックスを分離し、セッションはモデル呼び出し/ツール呼び出し/結果の append-only ログとして保持されます。
- Vaults とオンデマンド取得によって認証情報をサンドボックスの外に置けるため、プロンプトインジェクションによる露出リスクを下げられます。
- コンテナが存在する前から推論を開始でき、ツールを使わないセッションはコンテナ起動自体を回避できるため、レイテンシが改善し得ます。
- 耐久性のあるセッションは、デバッグのタイムライン、Memory、Dreaming（セッション/メモリをレビューして将来のメモリをキュレーションするスケジュール処理）を可能にします。

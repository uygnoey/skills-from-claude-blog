## Instructions

Use this skill when you want a practical workflow for operating an AI-native startup across the four stages described in the post.

1. **Identify your stage**: Idea, MVP, Launch, or Scale.
2. **Write exit criteria** for the stage (what must be true to move on).
3. **Choose the right Claude product surface** for the task (Chat, Claude Cowork, Claude Code), based on whether you need reasoning, collaboration, or code changes.
4. **Turn recurring founder tasks into repeatable workflows** so progress doesn’t depend on constant founder attention.

## Examples

- Idea stage: ask Claude to help structure a customer discovery plan and a competitive landscape map.
- MVP stage: use Claude Code to implement the smallest safe slice, with basic security and architecture guardrails.
- Launch stage: set up a lightweight measurement framework to distinguish real product-market fit from early hype.

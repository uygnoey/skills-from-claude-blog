# Claude Code 다이내믹 워크플로: 효과적으로 사용하는 방법

## 다이내믹 워크플로란?
다이내믹 워크플로는 Claude Code에서 Claude가 오케스트레이션 스크립트를 생성해 한 세션 안에서 수십~수백 개 서브에이전트를 병렬로 조율하고, 결과가 사용자에게 전달되기 전에 스스로 검증하는 기능입니다.

## 언제 쓰고(언제 피할까)
다이내믹 워크플로가 좋은 경우:
- 작업 범위가 매우 넓을 때(파일/모듈/미확인 영역이 많음)
- 큰 코드베이스 전반의 발견(discovery)·리뷰가 필요할 때
- 커밋 전에 계획을 다각도로 스트레스 테스트하고 싶을 때

피하거나 더 작은 범위로 시작할 경우:
- 이미 범위가 잘 정의되어 한 번의 집중 패스로 끝낼 수 있을 때
- 토큰 비용에 민감할 때(워크플로는 사용량이 크게 늘 수 있음)

## 권장 설정
- 최상의 경험을 위해 **auto mode** 사용을 권장합니다.
- 비용/효과 감을 잡기 위해 처음에는 **스코프가 작은 작업**으로 시작하세요.

## 시작 방법 2가지
1. Claude에게 직접 워크플로 생성을 요청합니다.
2. effort 메뉴에서 **ultracode**(effort xhigh)를 켜서 Claude가 필요 시 자동으로 워크플로를 사용하게 합니다.

## 실행 중 기대할 점
- 워크플로가 처음 트리거될 때, Claude Code가 실행될 항목을 보여주고 확인을 요청합니다.
- 워크플로는 장시간(수시간~수일) 실행을 전제로 합니다.
- 진행 상황이 저장되어 중단되더라도 재개할 수 있습니다.

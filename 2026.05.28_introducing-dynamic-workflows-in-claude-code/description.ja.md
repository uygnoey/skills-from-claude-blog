## この記事について
この記事は Claude Code の **Dynamic Workflows（動的ワークフロー）** を紹介します。Claude が 1 セッション内で数十〜数百のサブエージェントを並列に調整するオーケストレーションスクリプトを書き、ユーザーに届く前に自分でチェックできる実行方式です。

## どんなときに役立つか
一度のパスでは難しい大きな作業（特に複雑／レガシーなコードベース）で有用です。例:
- サービス全体にまたがるバグ調査
- 数百ファイルに触れるマイグレーション
- コミット前に多角的に検証したい計画

## 主なポイント
- **並列**かつ **長時間（数時間〜数日）** の実行を想定しています。
- 実行中に進捗が保存されるため、中断しても再開できます。
- ワークフローが初めてトリガーされる際、Claude Code が実行内容を表示し確認を求めます。
- Claude に直接ワークフロー作成を依頼するか、**ultracode**（effort xhigh）を有効にして必要に応じて Claude が自動で使うようにできます。
- 通常のセッションよりもトークン消費が大きくなる可能性があります。

## 이 글이 뭔가요
이 글은 Claude Code의 **다이내믹 워크플로(dynamic workflows)** 를 소개합니다. Claude가 한 세션 안에서 수십~수백 개의 서브에이전트를 병렬로 조율하는 오케스트레이션 스크립트를 작성하고, 결과가 사용자에게 도달하기 전에 스스로 점검할 수 있는 실행 방식입니다.

## 언제 유용한가요
한 번의 패스로 처리하기 어려운 큰 작업(특히 복잡하거나 레거시 코드베이스)에서 유용합니다. 예:
- 서비스 전체 범위의 버그 헌팅
- 수백 개 파일을 건드리는 마이그레이션
- 커밋 전에 여러 관점에서 검증하고 싶은 계획

## 핵심 포인트
- 워크플로는 **병렬 처리**와 **장시간 실행(수시간~수일)** 을 전제로 설계되었습니다.
- 실행 중 진행 상황이 저장되어, 중단되더라도 이어서 재개할 수 있습니다.
- 워크플로가 최초로 트리거될 때, Claude Code가 실행될 항목을 보여주고 확인을 요청합니다.
- Claude에게 직접 워크플로 생성을 요청하거나, **ultracode**(effort xhigh)를 켜서 Claude가 필요 시 워크플로를 자동으로 사용하게 할 수 있습니다.
- 일반 세션 대비 토큰 사용량이 크게 늘 수 있습니다.

## 번들 리소스
- Skill: 다이내믹 워크플로 사용 시점 판단과 워크플로 요청을 구조화하는 가이드.

## この記事について
この記事は、Claude Opus 4.6 と Claude Sonnet 4.6 において、100万トークン（1M）のコンテキストウィンドウが Claude Platform で一般提供（GA）になったことを告知します。

## どんなときに役立つか
短いコンテキスト制限を回避するための追加実装（要約、コンテキストのクリア等）を行わずに、非常に大きな入力を1回のリクエストで分析・参照したい場合に有用です（大規模コードベース、長い契約書、大きな差分、長時間動作するエージェントのトレース、大きなPDF、データセット、多数の画像など）。

## 主なポイント
- Opus 4.6 と Sonnet 4.6 で 1M コンテキストが GA。
- 1M 全域で標準料金が適用（ロングコンテキスト追加料金なし）。
- 1M 全域で標準のレートリミットが適用。
- 1回のリクエストで最大 600 枚の画像または 600 ページの PDF に対応（従来 100 から増加）。
- 200K トークン超のリクエストは beta ヘッダー不要（既に送っている場合は無視され、修正不要）。
- Opus 4.6 を利用する Claude Code の Max／Team／Enterprise でも 1M コンテキストが利用可能。

## 이 글이 뭔가요
이 글은 Claude Opus 4.6과 Claude Sonnet 4.6에서 100만 토큰(1M) 컨텍스트 창이 Claude Platform에서 정식(GA)으로 제공된다는 소식을 다룹니다.

## 언제 유용한가요
짧은 컨텍스트 한계를 우회하기 위한 별도의 엔지니어링(요약, 컨텍스트 비우기 등) 없이, 매우 큰 입력을 한 번의 요청에서 분석·참조해야 할 때 유용합니다(대규모 코드베이스, 긴 계약서, 큰 diff, 장기 실행 에이전트 트레이스, 대형 PDF, 데이터셋, 다량 이미지 등).

## 핵심 포인트
- Opus 4.6 및 Sonnet 4.6에서 1M 컨텍스트가 GA입니다.
- 전체 1M 구간에 표준 가격이 적용됩니다(롱컨텍스트 프리미엄 없음).
- 전체 구간에 표준 레이트 리밋이 적용됩니다.
- 요청당 최대 600장의 이미지 또는 600페이지의 PDF를 처리할 수 있습니다(기존 100에서 증가).
- 200K 토큰 초과 요청은 beta 헤더 없이 동작합니다(이미 beta 헤더를 보내고 있다면 무시되며 수정이 필요 없습니다).
- Opus 4.6 기준으로 Claude Code의 Max/Team/Enterprise에서도 1M 컨텍스트가 제공됩니다.

## 번들 리소스
- Skill: 1M 컨텍스트 모델을 효과적으로 활용하기 위한 가이드(유즈케이스 프레이밍, 운영상 고려사항).

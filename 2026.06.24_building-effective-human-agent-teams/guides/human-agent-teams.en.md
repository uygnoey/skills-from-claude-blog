# Human–agent team operating guide

## What this guide is
A practical guide for adopting human–agent teamwork as a repeatable operating model.

## How to use it
1. Choose a security boundary for the work.
2. Make key context public and searchable within that boundary.
3. Define a roster of humans and agents with clear ownership and tool access.
4. Set a north star goal.
5. Add verification and expand autonomy gradually.

## Checklist
### Public-by-default within the boundary
- Channels are public by default (inside the boundary).
- Decisions are captured in channels/docs/notes consistently.
- Documentation is written so agents can find and reuse it.

### Roles and tools
- Every human and agent has a written scope.
- Everyone works in shared threads to reduce handoffs.
- Tool access matches the responsibilities of each role.

### North star
- The north star is written, ambitious, and tied to business goals.
- Agents that may propose workstreams are explicitly named.
- High-fidelity human time is protected.

### Trust and autonomy
- Manual review happens early.
- A verifier step exists.
- Verification rubrics exist for key outputs.
- Autonomy expands by task type after repeated success.

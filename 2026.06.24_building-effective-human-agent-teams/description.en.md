## What is this post?
This post explains lessons from Anthropic’s internal experiments building “multiplayer” teams where humans and AI agents collaborate in shared workspaces.

## When is it useful?
Use it when you’re moving from ad-hoc, one-off AI usage to an operating model where agents can take ongoing responsibility for parts of a team workflow (triage, research, reporting, execution) under clear boundaries.

## Key points
- Work in public and give agents broad context (public-by-default channels, searchable docs, consistent decision capture).
- Define roles and provide the right tools (clear ownership, shared threads, explicit scopes).
- Set a “north star” goal so agents can be proactive without constant direction.
- Build trust over time by expanding autonomy only after repeated, verifiable successes.
- Agents participate best when they have persistent memory, safe credentials not tied to individuals, and broad access to information.

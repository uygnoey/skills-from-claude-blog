# Claude のコンピューター／ブラウザー利用のベストプラクティス

## この記事について
Claude の「computer use」「browser use」を使って UI 自動化（クリック／入力）を構築する際に、信頼性を高めるための実践的な推奨事項をまとめた記事です。スクリーンショットのリサイズ、座標の扱い、プロンプト構成、安定性／コストの調整、安全対策、長時間セッションのコンテキスト管理が主題です。

## どんなときに役立つか
- スクリーンショットを使って UI を操作するエージェントを実装するとき
- クリック精度が不安定（オフセット、ニアミス、別要素をクリック）なとき
- ダウンスケール、ズーム、thinking effort、キャッシュ、コンパクションの運用指針が必要なとき
- オープン Web 上で動作するエージェントのプロンプトインジェクション対策を設計するとき

## 主なポイント
- スクリーンショットは送信前にダウンスケールし、既定は 1280×720（Opus 4.7 は 1080p 開始を推奨）。
- リサイズした場合は、モデルが返した座標を実画面（ネイティブ）座標へ再スケーリングしてからクリックを実行する。
- message の content 配列では、テキスト指示を画像ブロックより前に置く。
- 小さなターゲットにはズームやキーボード操作を使い、極端に低い解像度は避ける。
- モデル／用途に合わせて thinking effort を調整し、計画（推論）と機械的操作を分離するオーケストレーター＋サブエージェントも検討する。
- Web コンテンツは信頼せず、内蔵の分類器、権限スコープの縮小、行動ログ、不可逆操作前の人間確認を基本にする。
- 長時間エージェントではキャッシュブレークポイント、古いスクリーンショットの整理、サーバー側コンパクションを活用する。

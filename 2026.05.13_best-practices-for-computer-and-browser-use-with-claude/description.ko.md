# Claude의 컴퓨터/브라우저 사용 모범 사례

## 이 글이 뭔가요
Claude로 “컴퓨터 사용(computer use)” 및 “브라우저 사용(browser use)” 자동화를 만들 때 신뢰성을 높이기 위한 실전 권장사항을 정리한 글입니다. 스크린샷 리사이징, 좌표 처리, 프롬프트 구성, 안정성/비용 튜닝, 안전 대책, 장기 실행 컨텍스트 관리가 핵심 주제입니다.

## 언제 유용한가요
- 스크린샷 기반 UI 자동화(클릭/타이핑)를 구현할 때
- 클릭이 빗나가거나(오프셋), 근처를 누르거나, 다른 요소를 누르는 문제가 있을 때
- 다운스케일, 줌, thinking effort, 캐시, 컴팩션 등 장시간 세션 운영 기준이 필요할 때
- 오픈 웹에서 동작하는 에이전트의 프롬프트 인젝션 대응을 설계할 때

## 핵심 포인트
- 스크린샷은 전송 전에 다운스케일하고, 기본값으로 1280×720을 권장합니다(Opus 4.7은 1080p 시작을 권장).
- 스크린샷을 리사이즈했다면, 모델이 반환한 좌표를 실제 화면 해상도로 다시 스케일링한 뒤 클릭을 실행해야 합니다.
- 메시지의 content 배열에서 텍스트 지시문을 이미지 블록보다 앞에 배치합니다.
- 작은 타겟은 줌 또는 키보드 내비게이션을 활용하고, 너무 낮은 해상도는 피합니다.
- 모델/상황에 맞춰 thinking effort를 조정하고, 계획(추론)과 실행(기계적 클릭)을 분리하는 오케스트레이터+서브에이전트 패턴도 고려합니다.
- 웹 콘텐츠는 신뢰하지 말고, 내장 프롬프트 인젝션 분류기 활용, 권한 범위 축소, 행동 로깅, 되돌릴 수 없는 작업 전 사용자 확인을 기본으로 둡니다.
- 장기 실행 에이전트는 캐시 브레이크포인트, 오래된 스크린샷 정리, 서버 사이드 컴팩션 등을 활용합니다.

## 번들 리소스
이 폴더에는 다음이 포함됩니다:
- 체크리스트와 구현 가이드를 담은 Agent Skill
- 스크린샷 준비, 좌표 스케일링, cache_control 배치, 스크린샷 정리용 컴패니언 스크립트
- 권장 기본값 및 “효과 없었던 최적화” 정리 레퍼런스

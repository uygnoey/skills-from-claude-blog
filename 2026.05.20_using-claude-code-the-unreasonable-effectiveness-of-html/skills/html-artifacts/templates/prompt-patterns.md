# Prompt patterns for HTML artifacts

## Options exploration grid

I’m not sure what direction to take <problem area>. Generate <N> distinctly different approaches—vary layout, tone, and density—and lay them out as a single HTML file in a grid so I can compare them side by side. Label each with the tradeoff it’s making.

## Implementation plan as HTML

Create a thorough implementation plan in an HTML file. Make it easy to read and digest. Include any mockups that help, show data flow, and include important code snippets I should review.

## PR review HTML explainer

Help me review this PR by creating an HTML artifact that describes it. I’m not very familiar with <area>, so focus on that. Render the actual diff with inline margin annotations, color-code findings by severity, and add whatever else is needed to convey the concept well.

## Design prototype tuning UI

I want to prototype <interaction>. Create an HTML file with several sliders and options so I can try different versions. Include a copy button to copy the parameters that worked well.

## Single-page technical explainer

I don’t understand how <system> works. Read the relevant code and produce a single HTML explainer page: a diagram (SVG if helpful), the 3–4 key code snippets annotated, and a “gotchas” section at the bottom. Optimize it for someone reading it once.

## Throwaway editor with export

I need to <edit task>. Make me an HTML file with a purpose-built editor UI for exactly this job. Add an export button that outputs the result as <JSON|prompt|Markdown|diff> so I can paste it back into Claude Code (or commit it to a file).

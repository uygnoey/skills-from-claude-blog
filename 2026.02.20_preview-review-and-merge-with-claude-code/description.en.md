# Preview, review, and merge with Claude Code

## What is this post?
This post announces new Claude Code Desktop capabilities that help you complete the development loop in one place: preview a running app, review local diffs before pushing, and monitor GitHub PR checks (optionally auto-fix and auto-merge).

## When is it useful?
When you want to shorten the time from ‘I changed code’ to ‘I confidently merged it’ without constantly context-switching between terminal, browser, and PR UI.

## Key points
- Preview: Start dev servers and preview the UI inside Claude Code Desktop, with Claude reading UI state and console logs.
- Review: Use the ‘Review code’ button to have Claude comment on your local diffs before you push.
- PR monitoring: Track CI status for GitHub pull requests in-app; optionally enable auto-fix for CI failures and auto-merge when checks pass.
- Continuity: Use `/desktop` to move a CLI session into the desktop app, or continue a desktop session on the web/mobile.

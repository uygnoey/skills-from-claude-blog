# Preview, review, and merge with Claude Code

## ¿De qué trata este post?
Este post anuncia nuevas capacidades de Claude Code Desktop para cerrar el ciclo de desarrollo en un solo lugar: previsualizar una app en ejecución, revisar diffs locales antes de hacer push y supervisar PRs de GitHub (con opciones de auto-fix y auto-merge).

## ¿Cuándo es útil?
Cuando quieres reducir el tiempo desde ‘cambié el código’ hasta ‘lo fusioné con confianza’ sin cambiar constantemente entre terminal, navegador y la interfaz del PR.

## Puntos clave
- Previsualización: inicia servidores de desarrollo y ve la UI dentro de Claude Code Desktop; Claude lee el estado visual y los logs de consola para iterar.
- Revisión: usa el botón ‘Review code’ para recibir comentarios sobre tus diffs locales antes de hacer push.
- Supervisión de PR: sigue el estado de CI de PRs de GitHub en la app; opcionalmente habilita auto-fix para fallos y auto-merge cuando todo pase.
- Continuidad: usa `/desktop` para llevar una sesión del CLI al escritorio, o continúa una sesión local en la web/móvil.

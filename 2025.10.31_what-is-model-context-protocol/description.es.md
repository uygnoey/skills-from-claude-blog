## ¿De qué trata este post?
Este post explica Model Context Protocol (MCP), un estándar abierto para conectar LLMs con herramientas y datos externos mediante una arquitectura cliente/servidor.

## ¿Cuándo es útil?
Es útil cuando quieres una forma estándar de integrar asistentes con muchas herramientas, evitando integraciones a medida para cada combinación modelo × herramienta.

## Puntos clave
- MCP se presenta como un conector universal (a menudo comparado con USB‑C) para integraciones AI‑herramienta.
- Un MCP Client (del lado del asistente) se conecta a MCP Servers (del lado de la herramienta/app).
- Está pensado para acceso seguro y escalable a herramientas y datos, con patrones locales y remotos.
- En Claude, los Connectors se basan en MCP para enlazar con servicios populares.
- El post remite a la documentación y a un registro para empezar.

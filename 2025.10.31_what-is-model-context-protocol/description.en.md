## What is this post?
This post explains Model Context Protocol (MCP), an open standard for connecting LLMs to external tools and data via a client/server architecture.

## When is it useful?
Use it when you want a standard way to integrate assistants with many tools (instead of building bespoke integrations for each model × tool pair).

## Key points
- MCP is positioned as a universal connector (often compared to USB-C) for AI-to-tool integrations.
- An MCP Client (on the assistant side) connects to MCP Servers (on the tool/app side).
- MCP is designed to support secure, scalable access to tools and data, including local and remote connection patterns.
- In Claude, MCP underpins Connectors that link to popular services.
- The post points readers to docs and a registry for getting started.

# Claude Code now supports artifacts

## What is this post?

This post announces Claude Code artifacts: live, interactive web pages that summarize and visualize in-progress work from a Claude Code session.

## When is it useful?

Use artifacts when you want to share the current state of engineering work (debugging, PR review, architecture exploration, dashboards, or release tracking) as a single link that updates as you work.

## Key points

- Artifacts are built from full session context (codebase, connectors, and conversation).
- Artifacts update in place as the session progresses; each publish creates a new version at the same link with version history.
- Artifacts are private by default, shareable within your organization, and cannot be made public.
- A gallery lets you browse and manage artifacts.
- Example prompts are provided for several roles (Legal, Privacy, Security, FinOps, Engineering, Design, Staff/Architecture, SRE, EM).

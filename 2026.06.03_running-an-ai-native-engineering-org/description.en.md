## What is this post?

This post explains how the Claude Code engineering team adapted its processes and org structure once agentic coding became the default way of working.

## When is it useful?

Use it when your team is adopting agentic coding and you need to update planning, context sharing, review practices, and role boundaries without losing quality or safety.

## Key points

- Replace long-range roadmaps with just-in-time planning, leaning more on PR discussions and prototypes.
- Shift context gathering from “ask the author” to “ask Claude first”, and then ask whether the step can be automated.
- Keep code review, but move human attention to trust boundaries, security-sensitive code, legal risk, and product taste.
- Expect role boundaries to blur (PMs and designers coding more; engineers doing more cross-functional work).
- Roll out new norms by defining a few non-negotiable principles and letting pods adapt details; track onboarding ramp time, PR cycle time, and Claude-assisted commits.

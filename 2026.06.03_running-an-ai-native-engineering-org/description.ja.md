## この記事について

この記事では、エージェント型コーディングが標準になった後、Claude Code のエンジニアリングチームがプロセスと組織運営をどのように変えたかを説明します。

## どんなときに役立つか

チームがエージェント型コーディングを導入し、計画・コンテキスト共有・レビュー・役割分担を品質と安全性を保ちながら見直す必要があるときに役立ちます。

## 主なポイント

- 長期ロードマップ中心から、PR 議論やプロトタイプに寄せた just-in-time 計画へ移行します。
- 「作者に聞く」ではなく「まず Claude に聞く」へコンテキスト収集を切り替え、手順自体を自動化できないかも検討します。
- コードレビューは維持しつつ、人は信頼境界・セキュリティに敏感なコード・法務リスク・プロダクトの嗜好といった重要箇所に集中します。
- 役割の境界が曖昧になる前提で運用します（PM/デザイナーがよりコードを書く、エンジニアが横断的な業務を担う）。
- 少数の非交渉原則を定め、細部は pod ごとに最適化させ、オンボーディングの立ち上がり時間／PR サイクルタイム／Claude 支援コミットを追跡します。

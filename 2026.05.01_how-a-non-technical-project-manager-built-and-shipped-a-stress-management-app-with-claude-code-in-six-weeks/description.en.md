## What is this post?
A case study of how Kostiantyn Vlasenko (a non-technical project manager) built and shipped *Respiro*, an iOS stress-management app, using Claude Code in under six weeks. Source: https://claude.com/blog/how-a-non-technical-project-manager-built-and-shipped-a-stress-management-app-with-claude-code-in-six-weeks

## When is it useful?
- When you want a realistic end-to-end example of shipping a real app with Claude Code as the primary development environment.
- When you’re considering a multi-agent/subagent approach for a complex app (architecture + implementation + review roles).

## Key points
- An initial prototype was built in ~72 hours for the Built With Opus 4.6 Claude Code Hackathon (February 2026), and the app reached App Store release in under six weeks.
- The author describes coordinating 15+ specialized subagents (e.g., TCA architect, Swift developer, Metal specialist, code reviewer) like a small team.
- Claude Code was used beyond coding: debugging via screenshots, App Store submission assistance, and marketing/growth tasks.

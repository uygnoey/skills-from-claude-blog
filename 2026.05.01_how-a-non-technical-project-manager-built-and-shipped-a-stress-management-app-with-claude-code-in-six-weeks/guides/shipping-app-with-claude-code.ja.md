# Claude Codeで実アプリを公開する（事例ベースのチェックリスト）

出典: https://claude.com/blog/how-a-non-technical-project-manager-built-and-shipped-a-stress-management-app-with-claude-code-in-six-weeks

## このガイドの内容
記事の事例から、プロトタイプ → サブエージェントによる役割分担 → 本番品質の強化 → App Store公開 → 指標で改善、という流れをチェックリスト化します。

## チェックリスト
1. **1文の目標**と制約（プラットフォーム、端末、スケジュール）を整理する。
2. 捨ててもよい **end-to-end の高速プロトタイプ** を先に作る。
3. **専門ロールを割り当てる**（アーキテクチャ、実装、レビュー、プラットフォーム専門）。
4. **本番品質を強化する**（クラッシュレポート、分析、スクリーンショットを使ったデバッグ）。
5. **公開準備**（提出物、ストア文言、マーケ草案）。
6. **計測して反復改善**（ファネル、リテンション、DAU/MAU）。

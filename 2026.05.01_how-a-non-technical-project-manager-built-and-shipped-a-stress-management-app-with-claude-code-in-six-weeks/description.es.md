## ¿De qué trata este post?
Un caso práctico sobre cómo Kostiantyn Vlasenko (un project manager no técnico) creó y publicó *Respiro*, una app de gestión del estrés para iOS, usando Claude Code en menos de seis semanas. Fuente: https://claude.com/blog/how-a-non-technical-project-manager-built-and-shipped-a-stress-management-app-with-claude-code-in-six-weeks

## ¿Cuándo es útil?
- Cuando quieres un ejemplo realista de extremo a extremo para publicar una app real usando Claude Code como entorno principal.
- Cuando estás considerando un enfoque multiagente/subagentes para una app compleja (arquitectura + implementación + revisión).

## Puntos clave
- Construyó un prototipo inicial en ~72 horas para el Built With Opus 4.6 Claude Code Hackathon (febrero de 2026) y llegó a la publicación en App Store en menos de seis semanas.
- Describe coordinar más de 15 subagentes especializados (por ejemplo, arquitecto de TCA, desarrollador Swift, especialista en Metal, revisor de código) como si fueran un equipo pequeño.
- Usó Claude Code también fuera del código: depuración con capturas de pantalla, ayuda para el envío a App Store y tareas de marketing/crecimiento.

## 이 글이 뭔가요
이 글은 Claude Code의 Code Review 기능을 소개합니다. 풀 리퀘스트(PR)가 열리면 여러 에이전트를 자동으로 투입해 버그·이슈를 찾고, PR에 요약 코멘트와 인라인 코멘트 형태로 리뷰 결과를 남깁니다.

## 언제 유용한가요
- 단순한 린터나 빠른 LLM 확인보다 더 깊고 촘촘한 리뷰 커버리지가 필요할 때
- 모든 PR에 일관된 리뷰 피드백을 자동으로 남기되, 최종 승인(merge)은 사람에게 남기고 싶을 때

## 핵심 포인트
- 모든 PR에 에이전트 팀이 투입되며, PR의 크기·복잡도에 따라 리뷰 깊이가 자동으로 조정됩니다.
- 결과는 "고신호" 요약 코멘트 1개 + 구체적 이슈에 대한 인라인 코멘트로 전달됩니다.
- 깊이를 우선하는 설계라, 가벼운 대안보다 비용이 더 들 수 있습니다.
- 이 기능이 PR을 승인하지는 않으며, 병합 여부는 사람이 결정합니다.

## 번들 리소스
- Skill: "에이전트 팀 기반 PR 자동 리뷰" (글에 나온 워크플로와 관리자/개발자 설정을 토대로 구성)

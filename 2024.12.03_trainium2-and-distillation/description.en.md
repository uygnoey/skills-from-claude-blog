## What is this post?
This post announces latency-optimized inference for Claude 3.5 Haiku on AWS Trainium2 in Amazon Bedrock, and preview support for model distillation in Amazon Bedrock.

## When is it useful?
Use it when you are choosing a deployment option for Claude workloads on AWS and want to understand the tradeoffs between a faster inference option and using distillation to improve a smaller model for specific tasks.

## Key points
- Claude 3.5 Haiku supports latency-optimized inference in Amazon Bedrock (public preview) and can deliver up to 60% faster inference speed.
- Amazon Bedrock Model Distillation can transfer task performance from a larger “teacher” model to a smaller “student” model by generating synthetic data, training/evaluating, and hosting the distilled model.
- The post also notes a price reduction for Claude 3.5 Haiku.

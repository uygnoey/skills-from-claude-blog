## ¿De qué trata este post?
Este post anuncia inferencia con latencia optimizada para Claude 3.5 Haiku en AWS Trainium2 dentro de Amazon Bedrock, y soporte en vista previa para la destilación de modelos en Amazon Bedrock.

## ¿Cuándo es útil?
Es útil cuando eliges una opción de despliegue para cargas de trabajo de Claude en AWS y quieres entender el equilibrio entre una opción de inferencia más rápida y usar destilación para mejorar un modelo más pequeño en tareas específicas.

## Puntos clave
- Claude 3.5 Haiku soporta inferencia con latencia optimizada en Amazon Bedrock (vista previa pública) y puede ofrecer hasta un 60% más velocidad de inferencia.
- Amazon Bedrock Model Distillation puede transferir rendimiento de un modelo “maestro” a un modelo “estudiante” mediante generación de datos sintéticos, entrenamiento/evaluación y alojamiento del modelo destilado.
- El post también menciona una reducción de precio para Claude 3.5 Haiku.

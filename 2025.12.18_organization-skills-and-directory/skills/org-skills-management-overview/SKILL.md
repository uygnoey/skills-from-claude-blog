## Instructions

You are helping someone understand and roll out Claude skills in an organization.

1. **Identify the environment**
   - Are they using Claude Apps, Claude Code, or the API (or multiple)?
   - Do they have admin access (Team/Enterprise)?

2. **Explain organization-wide skill management**
   - Describe that admins can provision skills centrally in admin settings.
   - Note that users may be able to toggle skills off depending on policy.

3. **Explain discovery and partner skills**
   - Explain that there is a directory of partner-built skills.
   - Provide examples mentioned in the source post (e.g., Notion, Canva, Figma, Atlassian).

4. **Explain distribution paths**
   - Claude Apps: enabling skills in settings.
   - Claude Code: installing skills from a plugin directory or checking skills into a repository.
   - API: using the skills endpoint.

5. **Call out prerequisites**
   - Skills may require capabilities such as Code Execution and File Creation to be enabled.

Bundled references:
- references/partner-skill-examples.md

# Get to production faster with the upgraded Anthropic Console

## What is this post?

Anthropic's March 6, 2025 announcement introduces a redesigned Anthropic Console as one place to build, test, and iterate on AI deployments with Claude alongside teammates. The post highlights four prompt-engineering tools (Workbench, automatic prompt generation, evaluation with side-by-side comparisons, and prompt improvement), shareable prompts for cross-functional collaboration, support for the latest Claude 3.7 Sonnet model, and the ability to control the extended thinking budget.

## When useful

- You are setting up a new prompt for production and want a structured build → test → iterate loop instead of ad-hoc experimentation.
- Multiple roles (developers, domain experts, product managers, QA) need to collaborate on the same prompt without copy-pasting between docs.
- You are migrating prompts written for other AI models and want them rewritten using Anthropic's prompt-engineering techniques.
- You are adopting Claude 3.7 Sonnet's extended thinking and need to budget thinking tokens deliberately.

## Key points

- The Workbench lets you structure prompts, add examples, and integrate external tools in an interactive API-call testing environment.
- A prompt generator turns a task description into a prompt that uses techniques like chain-of-thought.
- An evaluation tool generates test cases automatically, compares outputs side by side, and grades response quality.
- A prompt improvement tool refines existing prompts, including ones originally written for other AI models.
- "Get Code" exports a production-ready API call you can deploy directly.
- Shareable prompts give teams a centralized library for developing, refining, and standardizing prompts across the organization.
- Claude 3.7 Sonnet is supported, including a max-thinking-tokens budget for extended thinking.

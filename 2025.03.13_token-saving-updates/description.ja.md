## この記事について
この記事は、Anthropic API における Claude 3.7 Sonnet のトークン使用量を減らし、スループットを向上させるためのアップデート（キャッシュを考慮したレート制限、より簡単になったプロンプトキャッシュ動作、トークン効率の良いツール利用、そして新しい `text_editor` ツール）を紹介します。

## どんなときに役立つか
- ドキュメント、コードベース抜粋、ポリシーなど「長くてほぼ固定」のコンテキストを繰り返し送っていて、コストやレイテンシを下げたいとき
- 入力トークン分あたり制限（ITPM）で詰まっており、プロンプトキャッシュでスループットを上げたいとき
- tool calling のワークフローで出力トークンが大きくなりがちで、削減したいとき

## 主なポイント
- プロンプトキャッシュは、長いプロンプトでコスト（最大90%）とレイテンシ（最大85%）を削減できます。
- Anthropic API の Claude 3.7 Sonnet では、キャッシュから読み取ったトークンは Input Tokens Per Minute（ITPM）制限にカウントされなくなりました。
- プロンプトキャッシュが簡単になりました。キャッシュのブレークポイントを設定すると、Claude は過去にキャッシュされた最長のプレフィックスを自動で読み取ります。
- トークン効率の良いツール利用（ベータ）は、出力トークン消費を最大70%（平均約14%）削減できます。
- `text_editor` ツールは、コードやドキュメントなどテキストの特定部分に対するターゲット編集を可能にし、トークンとレイテンシの削減に役立ちます。

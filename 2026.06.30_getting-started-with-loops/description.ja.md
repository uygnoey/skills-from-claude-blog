## この記事について
この記事は、Claude Code チームが定義する **エージェンティック・ループ（agentic loops）**、すなわち **停止条件を満たすまで作業サイクルを繰り返す**考え方を解説します。

また、**ターンベース**の作業から **ゴールベース**、**時間ベース**、**プロアクティブ**なループへ発展させる流れと、それぞれが有効な場面を整理します。

## どんなときに役立つか
- 単発の対話セッションで十分か、検証可能なゴールループが必要か、定期ループが必要か、あるいは完全自動のルーティンが必要か判断したいとき
- 長時間の自動化で信頼性（自己検証）を高め、コスト/トークン使用量を管理したいとき

## 主なポイント
- まずは最もシンプルな方法から。すべてのタスクに複雑なループは不要です。
- 明確な **停止条件** を定義し、ゴールループでは **ターン上限** も明示します。
- 主観的判断ではなく、テスト/スコア/しきい値のような **決定的（deterministic）で検証可能な成功条件** を優先します。
- 品質向上のために Claude が結果を **検証** できる手段（ツール、スキル、定量チェック）を用意し、必要なら第二のエージェントでレビューします。
- トークン管理は、適切な primitive/model の選択、拡大前のパイロット実行、決定的作業のスクリプト化で行います。

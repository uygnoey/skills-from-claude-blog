# Claude Cowork を始めるためのベストプラクティス

## この記事について
この記事は、Claude Cowork をいつ使うべきか（チャットや Claude Code の代わりに）を整理し、Cowork で作業を委任する際に成果の質を上げるチェックリストと、開始時に使えるプロンプトを紹介します。

## どんなときに役立つか
既存の業務コンテキスト（ファイルや接続済みアプリ）の中で Claude に作業させ、誰かが開いて共有・提示・再利用できる成果物（deliverable）を作りたいときに役立ちます。特に、複数ステップで繰り返し発生するタスクに向きます。

## 主なポイント
- メール、資料、スプレッドシート、ドキュメント、会議、要約などの非技術系のナレッジワークには Claude Cowork が適していると述べています。
- 複数ステップ、複数ファイル／ファイル形式、複数アプリにまたがる作業なら Cowork を勧めています。
- 目安：数回のやり取りで済む質問／ブレスト／壁打ちはチャット、成果物が必要なら Cowork。
- まずは自分がよく分かっている「実際の仕事」から始め、品質を素早く評価できるようにする。
- 十分な文脈を渡し、望む結果（成果物）を明確にする。
- 開始前に、依頼内容の復唱と確認質問をするよう Claude に指示する。

## 同梱リソース
- プロンプトテンプレート: “Before we begin, repeat my ask back to me so we're aligned, then ask me as many clarifying questions as you have.”
- チェックリスト: タスクが「Cowork 向き」か判断する 5 つのサイン
- Cowork のタスク例（例: 毎日のブリーフィング、予算ペーシングのダッシュボード、レポーティング）

# Best practices for getting started with Claude Cowork

## What is this post?
This post explains when to use Claude Cowork (instead of chat or Claude Code) and gives a concrete checklist and starter prompt for getting good results when delegating work in Cowork.

## When is it useful?
Use it when you want Claude to work inside your existing work context (files and connected apps) and produce a deliverable someone will open, present, share, or reuse—especially for multi-step, repeatable tasks.

## Key points
- Prefer Claude Cowork for non-technical knowledge work that spans emails, docs, decks, spreadsheets, meetings, and summaries.
- Choose Cowork when the task is multi-step, touches multiple files/file types, or touches more than one app.
- A quick rule of thumb: chat for a few exchanges (questions/brainstorm/gut check), Cowork for deliverables.
- Start with a real task you already know well so you can quickly judge quality.
- Provide rich context and explicitly state the outcome (the deliverable).
- Ask Claude to repeat the request and ask clarifying questions before it begins.

## Bundled resources
- Prompt template: “Before we begin, repeat my ask back to me so we're aligned, then ask me as many clarifying questions as you have.”
- Checklist: five signals a task is “Cowork-shaped”.
- Examples of Cowork tasks (e.g., daily briefing, budget pacing dashboard, reporting).

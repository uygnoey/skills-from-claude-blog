# Buenas prácticas para empezar con Claude Cowork

## ¿De qué trata este post?
Este post explica cuándo conviene usar Claude Cowork (en lugar de chat o Claude Code) y ofrece una lista de verificación y un prompt inicial para obtener mejores resultados al delegar trabajo en Cowork.

## ¿Cuándo es útil?
Es útil cuando quieres que Claude trabaje dentro de tu contexto de trabajo existente (archivos y apps conectadas) y produzca un entregable que alguien abrirá, presentará, compartirá o reutilizará; especialmente para tareas de varios pasos y repetibles.

## Puntos clave
- Recomienda Claude Cowork para trabajo de conocimiento no técnico: correos, presentaciones, hojas de cálculo, documentos, reuniones y resúmenes.
- Sugiere Cowork cuando la tarea es de varios pasos, toca varios archivos/tipos de archivo, o usa más de una app.
- Regla rápida: chat para pocas interacciones (preguntas/lluvia de ideas/validación), Cowork para entregables.
- Empezar con una tarea real que ya conozcas bien para poder evaluar rápido la calidad.
- Aportar contexto rico y definir explícitamente el resultado (el entregable).
- Pedir a Claude que repita la solicitud y haga preguntas de aclaración antes de empezar.

## Recursos incluidos
- Plantilla de prompt: “Before we begin, repeat my ask back to me so we're aligned, then ask me as many clarifying questions as you have.”
- Checklist: cinco señales de que una tarea es adecuada para Cowork.
- Ejemplos de tareas en Cowork (p. ej., briefing diario, panel de pacing de presupuesto, reporting).

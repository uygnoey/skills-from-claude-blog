# Claude Cowork 시작을 위한 모범 사례

## 이 글이 뭔가요
이 글은 Claude Cowork를 언제 써야 하는지(채팅이나 Claude Code 대신) 기준을 정리하고, Cowork에서 일을 위임할 때 결과 품질을 높이는 체크리스트와 시작용 프롬프트를 제공합니다.

## 언제 유용한가요
파일과 연결된 앱 등 ‘이미 존재하는 업무 맥락’ 안에서 Claude가 작업을 진행하고, 누군가가 열어보고 공유하거나 발표·재사용할 수 있는 산출물(deliverable)을 만들어야 할 때 유용합니다. 특히 여러 단계로 이루어지고 반복되는 업무에 적합합니다.

## 핵심 포인트
- 이메일, 문서, 슬라이드, 스프레드시트, 회의 정리 같은 비기술 지식노동은 Claude Cowork가 적합하다고 설명합니다.
- 여러 단계로 진행되거나, 여러 파일/파일 형식을 다루거나, 둘 이상의 앱을 건드리는 작업이면 Cowork를 권합니다.
- 간단한 규칙: 몇 번의 대화로 끝나는 질문/브레인스토밍/검토는 채팅, 산출물이 필요하면 Cowork.
- 이미 잘 아는 ‘진짜 업무’부터 시작해 결과를 빠르게 평가하라고 조언합니다.
- 충분히 풍부한 맥락을 주고, 원하는 결과(산출물)를 명확히 말하라고 권합니다.
- 시작 전에 Claude가 요청을 다시 말하고 확인 질문을 하도록 지시하라고 제안합니다.

## 번들 리소스
- 프롬프트 템플릿: “Before we begin, repeat my ask back to me so we're aligned, then ask me as many clarifying questions as you have.”
- 체크리스트: 작업이 ‘Cowork에 맞는지’ 판단하는 5가지 신호
- Cowork 작업 예시(예: 매일 브리핑, 예산 페이싱 대시보드, 리포팅)

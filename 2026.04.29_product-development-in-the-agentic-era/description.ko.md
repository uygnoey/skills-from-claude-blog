# 이 글이 뭔가요

이 글은 제품 관리자가 Claude, Claude Code, Claude Managed Agents를 활용해 초기 탐색(리서치) 단계부터 반복 가능한 에이전트 워크플로를 실제로 배포하는 단계까지 어떻게 이어가는지 설명합니다.

# 언제 유용한가요

모호한 리서치를 빠르게 프로토타입으로 구체화하고, 이후 장시간 실행되는 클라우드 에이전트 세션으로 운영성 작업을 자동화하고 싶을 때 유용합니다.

# 핵심 포인트

- 문서로만 설계하지 말고 실제 API로 먼저 만들어 보며 추상화를 압력 테스트합니다.
- Claude/Claude Cowork는 개방형 탐색에, Claude Code는 ‘할 일(job to be done)’이 명확해져 에이전트로 코드화할 수 있을 때 사용합니다.
- PM용 예시 에이전트: 채택(Adoption) 분석, 개발자 피드백 모니터링, 데모 제작.
- Managed Agents 세션은 클라우드에서 실행되어 비동기 장기 작업과 실행 간 메모리(기록) 축적이 가능합니다.

# 번들 리소스

- PM 관점의 Managed Agents를 시작하기 위한 프롬프트 템플릿.
- 원문에 나온 3가지 에이전트 사용 사례를 정리한 Examples.
- 원문에 언급된 `claude-api skill` 호출 및 온보딩 프롬프트에 대한 레퍼런스 노트.

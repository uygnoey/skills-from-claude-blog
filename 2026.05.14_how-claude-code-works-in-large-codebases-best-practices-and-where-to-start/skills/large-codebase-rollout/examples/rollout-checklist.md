# Rollout checklist (derived from the post)

- [ ] Root `CLAUDE.md` is concise and includes only globally relevant conventions.
- [ ] Key subdirectories include their own `CLAUDE.md` for local rules.
- [ ] Generated/build/vendor directories are excluded via ignore settings.
- [ ] Deterministic checks (format/lint/test) are automated with hooks/scripts.
- [ ] LSP integrations are available for typed languages where symbol navigation matters.
- [ ] A DRI is assigned for configuration, permissions policy, and plugin governance.
- [ ] Config review cadence is set (every 3–6 months, plus after major model releases).
